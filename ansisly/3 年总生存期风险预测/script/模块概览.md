# CRC 总生存期风险预测流水线 — 模块概览

## 1. 系统简介

本系统是一套面向结直肠癌（CRC）多组学数据的总生存期（Overall Survival, OS）风险预测流水线。系统整合了临床信息、转录组表达谱、通路活性评分、体细胞突变等多模态特征，通过因果优先级筛选、条件生成对抗网络（GAN）数据增强、多种生存分析模型训练与锁定式内外部验证，构建并评估固定时间窗或全随访时间的 OS 风险预测模型。

系统设计遵循严格的防数据泄漏原则：
- 内部验证集不参与特征选择、GAN 训练、阈值选择或超参数搜索
- 外部验证队列仅在模型锁定后使用
- GAN 合成样本仅出现在训练集中

## 2. 脚本关系与差异

| 脚本 | 行数 | `TAU_MONTHS` | 端点类型 | 与 3YearsOS.py 的关系 |
|------|------|-------------|---------|----------------------|
| `3YearsOS.py` | 3501 | 36.0 | 固定 36 个月 OS | **基础实现**，包含全部功能模块的完整参考版本 |
| `1YearOS.py` | 3548 | 12.0 | 固定 12 个月 OS | 3YearsOS 的副本，仅修改 `TAU_MONTHS=12.0` 及相关标签文字 |
| `2YearsOS.py` | 3546 | 24.0 | 固定 24 个月 OS | 3YearsOS 的副本，仅修改 `TAU_MONTHS=24.0` 及相关标签文字 |
| `UnlimitedOS.py` | 1095 | N/A | 全随访时间/事件 | 通过 `importlib` 动态加载 3YearsOS 作为 `base` 模块，重写端点构建、预测和评估逻辑 |

> **设计模式说明**：`1YearOS.py`、`2YearsOS.py`、`3YearsOS.py` 三者共享几乎完全相同的代码结构，通过模块级常量 `TAU_MONTHS` 控制时间窗长度。`UnlimitedOS.py` 采用继承式扩展，复用 `base` 模块的工具函数并替换端点和评估逻辑。

## 3. 核心模块概览

以下模块描述以 `3YearsOS.py` 为基准。各固定时间窗脚本（1/2/3 年）中的同名函数行为一致，仅时间窗参数不同。

### 3.1 PipelineConfig — 全局配置

| 项目 | 说明 |
|------|------|
| **核心职责** | 使用 `@dataclass` 集中管理全部流水线超参数，包括路径配置、特征选择上限、GAN 训练参数、随机种子等 |
| **流程位置** | 基础设施层，贯穿全流程 |
| **主要对外接口** | `PipelineConfig` 类的字段即接口；`build_arg_parser()` 和 `config_from_args()` 提供 CLI → 配置转换 |
| **关键参数** | `tau_months`（时间窗）、`max_expression_features`（表达特征上限 300）、`max_causal_features`（因果特征上限 50）、`gan_aug_ratio_candidates`（GAN 增强比例候选）|

### 3.2 AuditLog / QuietAuditLog — 审计日志

| 项目 | 说明 |
|------|------|
| **核心职责** | 记录流水线各步骤的执行状态（OK/WARN/SKIP/FAIL），提供可追溯的运行日志。`QuietAuditLog` 抑制控制台输出，用于 GAN 交叉验证内部 |
| **流程位置** | 基础设施层，贯穿全流程 |
| **主要对外接口** | `audit.add(step, status, message, **extra)` 记录条目；`audit.to_frame()` 导出为 DataFrame |

### 3.3 端点构建模块

| 项目 | 说明 |
|------|------|
| **核心职责** | 从原始临床表构建生存分析结局变量。包含固定时间窗端点（`add_fixed_36m_endpoint`）、IPCW 权重计算（`compute_ipcw_weights`）、伪观测值计算（`compute_pseudo_observations`）以及 Kaplan-Meier 估计工具函数 |
| **流程位置** | **端点构建** — 数据预处理之后的第一步 |
| **主要对外接口** | `add_fixed_36m_endpoint(clinical, tau)` → 含 `death_by_36m`、`death_by_36m_observed`、`early_censored_before_36m` 列的 DataFrame；`compute_ipcw_weights(endpoint, tau)` → 添加 `ipcw_weight_36m` 列；`compute_pseudo_observations(endpoint, tau)` → 添加 `pseudo_risk_36m` 列 |
| **关键函数** | `kaplan_meier_survival_at()` — 手动实现 KM 生存函数估计，支持批量查询时间点；`detect_event_time_columns()` — 自动检测 OS 时间/事件列；`coerce_event()` — 将多种格式的事件标签统一为 0/1 整数 |

### 3.4 临床预处理模块

| 项目 | 说明 |
|------|------|
| **核心职责** | 从临床表中选取数值列（AGE、WEIGHT 等）和分类列（SEX、AJCC 分期、RACE 等），构建 `ColumnTransformer` + `Pipeline`，数值列走中位数填充 + StandardScaler，分类列走众数填充 + OneHotEncoder |
| **流程位置** | **数据预处理** — 特征工程的第一步 |
| **主要对外接口** | `build_clinical_preprocessor(train_df)` → `(Pipeline, List[str])`；`transform_clinical_frame(pipe, df, output_names)` → 用于外部队列的兼容变换 |
| **辅助类** | `DenseFrameTransformer` — 将稀疏矩阵转为稠密 numpy 数组，确保下游模型兼容 |

### 3.5 多组学特征工程模块

| 项目 | 说明 |
|------|------|
| **核心职责** | 处理转录组表达谱、通路活性和体细胞突变三类多组学数据。包括矩阵转置与患者 ID 归一化、基于训练集方差的 Top-N 特征选择、MSigDB GMT 通路活性评分计算、基于训练集流行度的突变特征选择、特征矩阵标准化 |
| **流程位置** | **特征工程** — 在临床预处理之后，因果筛选之前 |
| **主要对外接口** | `matrix_patients_to_rows(path)` → 患者×基因表达矩阵；`select_top_variance_features(matrix, train_ids, max_features)` → 高方差基因列表；`compute_pathway_activity(expression, raw_root)` → 通路活性评分 DataFrame；`select_train_prevalent_binary_features()` → 流行突变特征列表；`fit_transform_feature_matrix()` → 标准化后的特征矩阵 + Pipeline |
| **数据源** | TCGA-COAD/READ RNA-seq（log2 TPM-like）、MSigDB GMT 通路集、TCGA 体细胞突变基因级矩阵 |

### 3.6 因果筛选模块

| 项目 | 说明 |
|------|------|
| **核心职责** | 对表达特征进行因果优先级排序。使用 AIPW（增强逆概率加权）估计平均处理效应（ATE），使用 CATE（条件平均处理效应）评估效应异质性，使用剂量-响应分析检验单调性，最终综合排名产生因果优先级分数 |
| **流程位置** | **因果筛选** — 在特征工程之后，模型训练之前 |
| **主要对外接口** | `run_causal_screening(feature_matrix, clinical_adjustment, endpoint_train, random_seed, max_features)` → 按 `causal_priority_score` 降序排列的 DataFrame |
| **关键函数** | `causal_aipw_binary_exposure()` — 交叉拟合 AIPW 估计器（倾向性模型: LogisticRegression，结局模型: RandomForestRegressor）；`causal_cate_summary()` — 双模型 CATE 差异的 SD/IQR 统计量；`dose_response_summary()` — 分位数分箱后线性回归斜率 + Spearman 相关 |
| **结局变量** | 3YearsOS 使用 `pseudo_risk_36m_clipped`（伪观测值）；UnlimitedOS 使用 `os_event_risk_score`（风险代理分数）|

### 3.7 GAN 数据增强模块

| 项目 | 说明 |
|------|------|
| **核心职责** | 在训练集上拟合条件生成模型，产生合成样本以缓解小样本和事件类不平衡问题。提供四种候选方法并自动择优：WGAN-GP（`ConditionalTabularGAN`）、生存感知 WGAN-GP（`SurvivalConditionalTabularGAN`）、特征空间 GAN（`FeatureSpaceGAN`）、SMOTE 插值（`SmoteLikeAugmenter`）。包含五维合成数据 QC 评估和基于训练集 CV 效用的比例/策略自动选择 |
| **流程位置** | **数据增强** — 在特征工程和因果筛选之后，模型训练之前 |
| **主要对外接口** | `select_train_only_gan_augmentation(X_train, endpoint_train, cfg, audit)` → `(X_aug, endpoint_aug, report)`；`train_only_gan_augmentation()` — 单次 GAN 拟合 + QC + 合成 |
| **QC 接口** | `synthetic_data_qc(X_real, X_syn, X_holdout)` → 含 KS 检验、Wasserstein 距离、PCA 方差相关性、DCR 比率、MIA 风险评估的字典 |
| **增强策略** | 四种采样策略：`balanced_event`（事件类平衡）、`risk_stratified`（风险分层）、`event_only`（仅增强事件类）、`overall`（整体采样）|
| **关键约束** | 仅在 TCGA_train_real 上拟合，合成样本不出现在内部验证或外部验证中 |

### 3.8 生存模型训练模块

| 项目 | 说明 |
|------|------|
| **核心职责** | 在多种特征集（仅临床、临床+表达、临床+因果、临床+通路、临床+突变）上训练多种生存分析模型：IPCW 加权 Logistic 回归、Cox PH（lifelines）、Coxnet（scikit-survival）、随机生存森林 RSF（scikit-survival）。对 GAN 增强后的数据额外训练增强版模型 |
| **流程位置** | **模型训练** — 在数据增强之后 |
| **主要对外接口** | `fit_weighted_logistic(X_train, endpoint_train, random_seed)` → LogisticRegressionCV；`fit_lifelines_cox(X_train, endpoint_train, penalizer)` → CoxPHFitter；`fit_sksurv_coxnet(X_train, endpoint_train)` → CoxnetSurvivalAnalysis；`fit_sksurv_rsf(X_train, endpoint_train, random_seed)` → RandomSurvivalForest |
| **预测接口** | `risk_from_model()` — 二分类模型概率输出；`predict_lifelines_risk()` — Cox PH 生存函数在 τ 处的 1-S(t)；`predict_sksurv_risk()` — sksurv 模型生存函数在 τ 处的 1-S(t) |
| **特征集** | `clinical_only`、`clinical_expression_topvar`、`clinical_causal_priority`、`clinical_pathway`、`clinical_somatic_mutation` |

### 3.9 模型评估模块

| 项目 | 说明 |
|------|------|
| **核心职责** | 对训练集和内部验证集上的模型预测进行多维评估。指标包括：IPCW 加权 AUC、IPCW 加权 Brier 分数、平均精度（AP）、Harrell C-index、Uno C-index（IPCW）、时间依赖 AUC、混淆矩阵（基于 Youden 最优阈值）|
| **流程位置** | **模型评估** — 伴随模型训练 |
| **主要对外接口** | `evaluate_36m_predictions(train_endpoint, eval_endpoint, risk, tau, threshold)` → 含全部指标的字典；`choose_training_threshold()` — 基于 ROC 曲线 Youden 指数选取最优阈值 |
| **辅助函数** | `uno_cindex_fallback()` — 手动实现 Harrell C-index（O(n²) 对遍历）；`bootstrap_ci()` — Bootstrap 置信区间 |
| **模型选择** | 基于内部验证集 Uno C-index、时间依赖 AUC、Brier 分数的综合排名选择主模型，外部队列不参与选择 |

### 3.10 可视化模块

| 项目 | 说明 |
|------|------|
| **核心职责** | 为选定模型在各队列上生成评估图表：时间依赖 ROC 曲线、校准曲线（预测 vs 观测分位数分箱）、Kaplan-Meier 风险分层生存曲线。每张图同时输出 PNG（300 dpi）和 TIFF（600 dpi）|
| **流程位置** | **可视化** — 在模型选择之后 |
| **主要对外接口** | `generate_prediction_figures(endpoint, risk, threshold, cohort_label, model_name, tau, figures_dir)` → 图表元数据列表 |
| **关键函数** | `plot_time_dependent_roc()` — IPCW 加权 ROC；`plot_calibration_curve()` — 分位数分箱校准图；`plot_km_risk_strata()` — 高/低风险组 K-M 曲线 + log-rank 检验 p 值 |

### 3.11 外部验证模块

| 项目 | 说明 |
|------|------|
| **核心职责** | 加载外部队列（CPTAC、GEO、HTAN、MSK 等），构建与训练集兼容的特征矩阵，对锁定主模型进行独立外部验证。同时输出所有候选模型的外部诊断指标（不参与模型选择）|
| **流程位置** | **外部验证** — 在主模型锁定之后 |
| **主要对外接口** | `load_external_cohorts(preprocessed_root, tau, audit)` → `{cohort_name: endpoint_df}`；`build_external_feature_matrix(cohort_name, endpoint, raw_root, ...)` → `(X_ext, compatibility_dict)` |
| **兼容机制** | 自动匹配外部队列可用的数据层（临床/表达/通路/突变），缺失层标记为 `not_available`。对缺失的临床列进行 NaN 填充，但多组学特征缺失时跳过该队列 |

### 3.12 流水线主控模块

| 项目 | 说明 |
|------|------|
| **核心职责** | 编排上述所有模块的执行顺序，管理输出目录结构（tables/models/reports/figures），生成运行清单（run manifest）、泄漏审计（leakage audit）和外部泛化诊断摘要 |
| **流程位置** | 顶层控制 |
| **主要对外接口** | `run_pipeline(cfg)` → 包含 run_dir、selected_primary_model、各输出路径的字典；`main(argv)` → CLI 入口 |
| **输出产物** | `dependency_report.tsv`、`run_manifest.json`、`feature_blocks_manifest.json`、`internal_validation_model_comparison.tsv`、`external_real_world_validation_metrics.tsv`、`figure_manifest.tsv`、`leakage_audit.json`、`pipeline_audit_log.tsv` 等 |

## 4. UnlimitedOS 特有模块

`UnlimitedOS.py` 不使用固定时间窗，而是利用全随访时间/事件对构建端点。

| 模块/函数 | 核心职责 |
|-----------|---------|
| `load_three_years_base()` | 通过 `importlib.util` 动态加载 `3YearsOS.py` 为 `base` 模块，复用其工具函数 |
| `add_unlimited_os_endpoint()` | 构建全随访 OS 端点：`os_event`（死亡=1/删失=0）、`log1p_time_months`、`os_event_risk_score` |
| `make_os_event_risk_proxy()` | 基于时间百分位排名和事件状态计算风险代理分数：`event * (1 - 0.5*time_rank) + (1-event) * 0.25*(1-time_rank)` |
| `predict_os_risk_score()` | 适配全时间范围的风险预测：Cox PH 使用 `predict_partial_hazard`，其他模型使用 `predict()` |
| `predict_survival_matrix()` | 预测生存函数矩阵 S(t|x)，支持在多个评估时间点上查询 |
| `evaluate_os_predictions()` | 使用 Harrell C-index + Uno IPCW C-index + Integrated Brier Score 评估，不计算固定时间窗 AUC |
| `plot_km_risk_strata_os()` | Unlimited 版 K-M 风险分层图（无 τ 标记线）|
| `plot_risk_distribution_by_event()` | 按死亡/删失状态展示风险分数分布箱线图 |
| `plot_followup_time_vs_risk()` | 随访时间 vs 风险分数散点图 |

## 5. 数据流概览

```
原始临床表 (TCGA OS endpoint TSV)
    │
    ▼
端点构建 ──→ 固定时间窗端点 + IPCW 权重 + 伪观测值
    │         (或 UnlimitedOS: 全随访时间/事件端点)
    ▼
数据分割 ──→ TCGA_train_real + TCGA_internal_validation_real
    │          (预定义 split 文件 或 分层随机分割)
    ▼
特征工程 ──→ 临床预处理 + 表达特征(Top方差) + 通路活性 + 体细胞突变
    │
    ▼
因果筛选 ──→ AIPW/CATE/剂量响应 → 因果优先级排名 (仅训练集)
    │
    ▼
GAN 增强 ──→ 多方法候选 → QC 评估 → 效用验证 → 最优增强方案 (仅训练集)
    │
    ▼
模型训练 ──→ 多特征集 × 多模型类型 × (原始/增强)
    │
    ▼
内部评估 ──→ Uno C-index / AUC / Brier / AP → 综合排名选择主模型
    │
    ▼
外部验证 ──→ 锁定主模型 → 外部队列独立评估 + 全候选模型诊断
    │
    ▼
输出产物 ──→ 指标表 / 模型文件 / 图表 / 审计报告 / 泄漏审计
```
